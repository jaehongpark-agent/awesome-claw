# Contributing to awesome-claw

Thanks for your interest in contributing! This list aims to be the most comprehensive, well-organized resource for the Claw ecosystem.

## Guidelines

### Adding a New Entry

1. **Search first** — Make sure the project isn't already listed.
2. **Pick the right category** — Place your entry in the most relevant section.
3. **Use the correct format:**

   For table entries:
   ```markdown
   | [Name](https://link) | Language | Stars | One-line description |
   ```

   For list entries:
   ```markdown
   - **Name** — Short description
   ```

4. **Keep descriptions concise** — One line, focused on what makes the project unique.
5. **Include a working link** — All links must point to a live page or repository.

### Star Counts

- Star counts are approximate and grouped into tiers (Tier 1: 10K+, Tier 2: 1K–10K, Emerging: <1K).
- When adding a self-hosted platform, include the approximate star count.
- Projects may be moved between tiers as they grow.

### Quality Standards

- The project should be **actively maintained** (commit activity within the last 6 months).
- The project should have a **clear README** or documentation.
- The project should be **relevant to the Claw ecosystem** (always-on AI agents, not one-shot scripts).
- Security-related entries should include **verifiable sources** (CVE IDs, vendor advisories, etc.).

### What We Don't Accept

- Projects that are forks with no meaningful changes
- Duplicate entries
- Self-promotional entries without substance
- Entries with broken or dead links

## Pull Request Process

1. Fork the repository.
2. Create a new branch (`git checkout -b add-project-name`).
3. Make your changes following the guidelines above.
4. Verify all links are working.
5. Submit a pull request with a clear description of what you're adding and why.

## Updating Existing Entries

- If a project has moved, been renamed, or archived, please submit a PR to update or remove it.
- Star count updates are welcome but not required for every PR.

## Questions?

Open an issue if you're unsure where something belongs or if you'd like to propose a new category.
